Height Normalization

Uniform Cap HeightScale every glyph so its height = Cap Height. Preserves aspect ratio.
Flip Y axisSVG y-axis points down; font y-axis points up. Required for most SVGs.

Side Bearings

Zero Side BearingsSet advance width = xMax, shift outline so xMin = 0. Correct for italic/script/display fonts where outlines fill the full glyph width. Turn OFF for upright fonts with intentional spacing.

Italic/script/display: use Zero. Upright/serif/sans: use Original or Custom. The kern algorithm works correctly either way — it reads the actual outline, not the advance width.

Touch Kerning

Analytical bezier kern — walks each glyph's actual path curves mathematically, no canvas rasterization. Samples right and left silhouettes at every Y level with sub-unit precision. Works correctly for any font type: italic, upright, script, display, variable-width. Asymmetric pairs (AB ≠ BA) computed independently.

How It Works

For each pair (A→B): cubic/quadratic bezier segments are walked parametrically, sampling x at each Y sub-unit row. K = max over all rows of (rightEdge_A − leftEdge_B − advWidth_A + gap). Taking the max ensures the tightest kern with no overlap at any Y. Both directions (A→B and B→A) are computed independently — correct asymmetry guaranteed. Positive kern (gap) values are also stored, needed for curved upright letters that visually appear too tight.
